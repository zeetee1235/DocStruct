# 리팩토링 요약

이번 정리는 실행 파이프라인을 모듈화하고, 중복/산재된 로직을 정리해 코드 가독성과 테스트 가능성을 높이는 데 초점을 맞췄습니다.

## 변경 요점
- **파이프라인 모듈화:** 문서 생성과 내보내기(export) 로직을 `pipeline` 모듈로 이동해 책임을 명확히 분리했습니다.
- **CLI 단순화:** `main.rs`에서는 구성(`PipelineConfig`)과 실행 함수 호출만 담당하도록 정리했습니다.
- **테스트 가능성 개선:** 디버그 정보 부착과 내보내기 로직을 함수로 분리해 테스트 코드에서 직접 검증할 수 있도록 했습니다.

## 정리 결과
- 파이프라인 생성/내보내기 흐름이 한눈에 보이도록 구조를 단순화했습니다.
- 중복되던 디버그 주입 로직을 한 곳에 모아 가독성과 유지보수성을 높였습니다.
- 단위 테스트에서 핵심 로직을 검증할 수 있게 되어 안정성을 확보했습니다.
